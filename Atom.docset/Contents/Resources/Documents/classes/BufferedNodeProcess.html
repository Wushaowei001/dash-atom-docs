    Class BufferedNodeProcess

    Defined in: src/buffered-node-process.coffee
    Inherits: BufferedProcess

    Overview

    Like BufferedProcess, but accepts a Node script as the command to run.

    This is necessary on Windows since it doesn't support shebang #! lines.

    Requiring in packages

    {BufferedNodeProcess} = require 'atom'

    ::constructor({command, args, options, stdout, stderr, exit}) Source

    Runs the given Node script by spawning a new child process.

    Parameters:

    • (Object) options — An Object with the following keys:
      • (String) command — The String path to the JavaScript script to execute.
      • (Array) args — The Array of arguments to pass to the script (optional).
      • (Object) options — The options Object to pass to Node's ChildProcess.spawn method (optional).
      • (Function) stdout — The callback Function that receives a single argument which contains the standard output from the command. The callback is called as data is received but it's buffered to ensure only complete lines are passed until the source stream closes. After the source stream has closed all remaining data is sent in a final call (optional).
      • (Function) stderr — The callback Function that receives a single argument which contains the standard error output from the command. The callback is called as data is received but it's buffered to ensure only complete lines are passed until the source stream closes. After the source stream has closed all remaining data is sent in a final call (optional).
      • (Function) exit — The callback Function which receives a single argument containing the exit status (optional).
